## Uso recomendado

1. Lee el enunciado completo antes de modelar.
2. Identifica entidades del dominio, atributos y relaciones.
3. Decide qué clases deberían existir antes de escribir código.
4. Si hay archivos `.txt`, define cómo se cargarán y qué estructura almacenará los datos.

El foco de estas ayudantías no es memorizar clases, sino practicar la traducción desde un problema escrito hacia un modelo de objetos coherente.
